# 第三章：数据 — 知识点总结

## 3.1 基本数据类型

### 各类型位宽

| 类型 | 最小位数 | 本平台（MinGW64） |
|------|----------|-------------------|
| `char` | 8 位 | 8 位（1 字节） |
| `short` | 16 位 | 16 位 |
| `int` | 16 位 | 32 位 |
| `long` | 32 位 | 32 位（Win）/ 64 位（Linux） |
| `long long` | 64 位 | 64 位 |

> ⚠️ `long` 在 Windows 和 Linux 上大小不同，嵌入式跨平台注意用 `sizeof()` 确认或用 `int32_t`。

### char 的三种形式

| 声明 | 范围 |
|------|------|
| `signed char a;` | -128 ~ 127 |
| `unsigned char a;` | 0 ~ 255 |
| `char a;` | 取决于编译器，MinGW 上等同 signed |

> `char`、`signed char`、`unsigned char` 是三种不同的类型。存数值时显式写 `signed`/`unsigned`。

### 科学计数法（e 记法）

```c
1e10   = 1 × 10^10 = 10000000000.0（double）
1e-3   = 0.001
2.5e6  = 2500000.0
```

- 带 `e` 或带 `.` 的字面量一律是浮点类型，由**格式**决定，不由计算结果决定
- 加后缀改变类型：`1e10f`（float）、`1e10L`（long double）

## 3.2 基本声明

### 声明的读法

C 语言的声明遵循"声明模仿使用"原则：

```c
int *p;       // *p 是一个 int → p 是指向 int 的指针
int a[10];    // a[0] 是 int → a 是 int 数组
int f();      // f() 返回 int → f 是返回 int 的函数
```

### 声明中的 * vs 表达式中的 *

```c
int *p = &a;       // 声明：* 表示 p 是指针，= 是初始化（把 a 的地址给 p）
*p = 10;           // 表达式：* 是解引用，去 p 存的地址改值
```

同一个 `*` 两种含义，分两行写不会混：

```c
int *p;      // 声明类型
p = &a;      // 赋值
```

### 指针基础

- **指针** = 存地址的变量。`p` 是指针（存地址），`*p` 是指针指向的数据
- `&a` = 取 a 的地址
- `*p` = 去 p 存的地址取东西（对应汇编的**间接寻址**）
- `int *p, q;` → p 是指针，q 是 int（声明中 `*` 只作用于紧邻的变量）

## 3.3 typedef

### typedef vs #define

| | `typedef` | `#define` |
|------|-----------|------------|
| 处理者 | 编译器 | 预处理器 |
| 实质 | 给类型起别名 | 纯文本替换 |
| 典型坑 | 无 | `#define PTR int*; PTR a, b;` → 只有 a 是指针 |

> 指针类型起别名必须用 `typedef`，别用 `#define`。

## 3.4 常量

### 指针常量

指针常量 = 地址本身是常量，不能变。三种常见形式：

| 形式 | 示例 |
|------|------|
| 空指针 | `NULL` / `(void *)0` |
| 数组名 | `arr` 不能重新赋值 |
| 字符串字面量 | `"hello"` 存在只读区 |

### 字符串常量

双引号括起来的字符序列，存在**只读区**，程序运行全程存在：

```c
char *s = "hello";     // 指向只读区，内容不能改
char arr[] = "hello";  // 在栈上复制了一份，能改
```

编译器自动在末尾加 `\0`：`"hello"` 占 6 字节。

## 3.5 作用域

| 作用域 | 范围 | 示例 |
|--------|------|------|
| 文件作用域 | 从定义处到文件末尾 | 全局变量、函数定义 |
| 代码块作用域 | 所在 `{}` 内部 | 局部变量 |
| 函数作用域 | 整个函数体内 | 仅 `goto` 标签 |
| 原型作用域 | 函数声明的 `()` 内 | `void f(int x);` 中的 x |

> 日常 99% 关心的是文件作用域和代码块作用域。

### 函数不能嵌套定义

```c
void outer() {
    void inner() { }   // ❌ C 语言不允许
}
```

函数全部写在文件作用域下，靠声明和调用配合。

---

## 3.6 链接属性

决定同一个名字在不同文件中能否指向同一实体：

| 属性 | 关键字 | 范围 |
|------|--------|------|
| 外部链接 | （默认） | 所有文件可见 |
| 内部链接 | `static` | 仅本文件可见 |
| 无链接 | （局部变量） | 仅当前 `{}` 可见 |

> 链接属性是被调用者自己的属性，和谁来调用无关。`main()` 调 `static` 函数 → 函数仍是内部链接。

---

## 3.7 存储类型

决定变量**存在哪里、活多久**：

| 关键字 | 存哪里 | 生命周期 |
|--------|--------|----------|
| `auto` | 栈 | 出了 `{}` 就死（默认，从不写） |
| `register` | CPU 寄存器（建议） | 出了 `{}` 就死 |
| `static` | 静态区 | 程序运行全程 |
| `extern` | 静态区（借别处的） | 程序运行全程 |

### static 的三种用法

| 位置 | 作用 | 改变的是 |
|------|------|----------|
| 局部变量前 | 变量活全程，只初始化一次 | 生命周期 |
| 全局变量前 | 仅本文件可见 | 链接属性 |
| 函数前 | 仅本文件可调用 | 链接属性 |

### extern

```c
extern int x;           // 声明：x 在别处定义，借来用
extern int x = 0;       // 有初始化 → 变成定义，不再"借"
```

- `extern` 声明的变量可以读写
- 不要在 `extern` 声明时赋初值

### 全局变量存储位置

全局变量都在**静态区**，与是否加 `static` 无关。`static` 只改变链接属性（能否被其他文件访问），不改变存储位置。

---

## 枚举类型（enum）

给整数起一组有意义的名字：

```c
enum color { RED, GREEN, BLUE };     // 0, 1, 2
enum week { MON = 1, TUE, WED };     // 1, 2, 3（自动递增）
```

枚举 vs `#define`：枚举有**分组语义**，让代码更可读。嵌入式常用于状态机：

```c
enum motor_state { STOP = 0, FORWARD, REVERSE, BRAKE };
```
