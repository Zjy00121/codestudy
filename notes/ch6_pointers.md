# 第六章：指针 — 知识点总结

## 6.1 内存和地址

### 左值和右值（5.3 补充纠正）

| | 左值 | 右值 |
|------|------|------|
| 定义 | 有持久内存位置，可取地址 `&` | 临时值，无固定地址 |
| 例子 | 变量、`*p`、`*(p+1)` | 字面量 `10`、`a+b`、`a++`、`++a` |

> **C 语言中 `++i` 和 `i++` 都不是左值**（C++ 中 `++i` 是左值）。修正此前笔记。

### 指针声明与初始化

- 声明指针时**不一定**要初始化，但建议初始化为 `NULL` 或有效地址
- 未初始化的指针存的是随机垃圾地址，解引用 = 未定义行为
- `int **pp` — 二级指针，多一个 `*` 就多一级间接寻址

### NULL 指针

- `#define NULL ((void*)0)` — 本质是地址 0
- 解引用 NULL → 段错误（操作系统保护地址 0）
- 函数返回指针时用 `NULL` 表示失败，调用方先判空再使用
- `NULL` 是指针值，不要赋给普通变量；`'\0'` 是空字符，两者不同

### 间接访问 `*`

- `*` 操作数必须是指针类型，编译报错如果用于非指针
- 强制转换整数为指针可以绕过：`*(int *)addr = 1;`（嵌入式日常操作）
- 地址必须合法（属于你、在生命期内、对齐），否则崩溃

### 指针的指针（二级指针）

声明规则 — 多一个 `*` 就多一级：
```c
int   a = 10;
int  *p = &a;     // 一级 — 指向 int
int **pp = &p;    // 二级 — 指向 int*
```

**多一级有什么用？** 要在函数里修改谁，就得传谁的地址：
```c
void push_front(Node **head, int val);  // 可能改头指针，传入 head 的地址
int main(int argc, char **argv);        // argv 就是二级指针
```

## 6.11 指针的表达式

### `*++cp` vs `*cp++`

| 表达式 | 等价写法 | 行为 |
|--------|----------|------|
| `*++cp` | `*(++cp)` | 指针先前进，再解引用 → 访问下一元素 |
| `*cp++` | `*(cp++)` | 先解引用（当前元素），指针再前进 |

**优先级**: 后置 `++` > 前置 `++` / `*`，前置 `++` 和 `*` 同级（右结合）。

### 四种组合对比

```c
int arr[] = {10, 20, 30, 40, 50};
int *cp = arr;    // cp → arr[0] = 10
```

| 表达式 | 等价写法 | 指针动了？ | 改了数据？ | 表达式值 |
|--------|----------|:--:|:--:|:----:|
| `++*cp` | `++(*cp)` | ✗ | ✓ 改 arr[0] | 新值 |
| `(*cp)++` | — | ✗ | ✓ 改 arr[0] | 旧值 |
| `++*++cp` | `++(*(++cp))` | ✓ 跳到 arr[1] | ✓ 改 arr[1] | 新值 |
| `++*cp++` | `++(*(cp++))` | ✓ 跳到 arr[1] | ✓ 改旧位置 arr[0] | 新值 |

> 以上四种全是右值，不能放 `=` 左边。
> `*cp++` 本身可以当左值 — `*(cp++)` 解引用得到真实内存位置。

### `(*cp)++` 和 `*cp++` 的区别

```c
(*cp)++   // 括号抱住 *cp → 对数据做后置 ++，改 arr[0]
*cp++     // *(cp++) → 对指针做后置 ++，读 arr[0]，指针移走
```

差一个括号，差之千里。

## 6.12 指针的运算

### 四种运算

| 运算 | 含义 | 规则 |
|------|------|------|
| `p + n` | 向后跳 n 个元素 | 实际位移 = `n × sizeof(*p)` |
| `p - n` | 向前跳 n 个元素 | 同上 |
| `p2 - p1` | 两指针间元素个数 | 必须同类型指针 |
| `p < q` / `p == q` | 地址比较 | 遍历边界判断 |

### 不能做的事

`p + q`, `p * q`, `p / q`, `p >> 1`, `p + 3.14` — 全不合法

### 指针运算自动缩放

```c
char  *c → c + 1 = 地址 + 1  (char=1字节)
short *s → s + 1 = 地址 + 2  (short=2字节)
int   *i → i + 1 = 地址 + 4  (int=4字节)
```

**二级指针的运算规则一模一样** — `sizeof(*pp) = sizeof(int*)`，`pp + 1` 跳过 8 字节（一个指针大小）。

## 6.9 指针常量

### `const` 修饰指针的三种形式

| 声明 | 指针可变？ | 可改数据？ | 判断诀窍 |
|------|:---:|:---:|------|
| `const int *p` | ✅ | ❌ | `const` 在 `*` 左边 → 约束内容 |
| `int const *p` | ✅ | ❌ | 同上，`const int` = `int const` |
| `int * const p` | ❌ | ✅ | `const` 在 `*` 右边 → 约束指针本身 |
| `const int * const p` | ❌ | ❌ | 两边都有，全锁 |

### C 中的"指针常量"

- **字符串字面量** `"hello"` → `char *`（存在只读区）
- **复合字面量** `(int[]){1,2,3}` → 匿名数组，退化后为 `int *`（栈上）
- **强制转换** `(int *)0x40020000` → 嵌入式日常操作
- 没有通用的 `&字面量` 语法，因为地址是运行时概念

## 数组 vs 指针

### 数组名在表达式中"退化"成指针

```c
int a[5];
int *p = a;      // a 退化成 &a[0]
func(a);         // 传参时退化
a + 1;           // 做运算时退化
```

三个不退化的例外：
```c
sizeof(a);       // 整个数组大小
&a;              // 指向整个数组的指针 (int (*)[5])
char s[] = "abc"; // 字符串初始化时拷贝
```

### `a[i]` 等价于 `*(a + i)`

中括号不是独立运算符，编译器内部就是按 `*(指针 + 偏移)` 实现的。

### 二维数组不退化出 `char **`

```c
char words[3][10];  // 退化成 char (*)[10]（指向 10 个 char 的数组的指针）
                    // 不是 char **！

char *words[];      // 这才退化出 char **
```

| | `char *words[]` | `char words[3][10]` |
|--|--|--|
| 存什么 | 3 个**指针** | 30 个**字符**（连续） |
| 退化后 | `char **` | `char (*)[10]` |

**`char **` 只在实际存了一组指针时用。**

## 练习函数总结

### Q_1 — 指针表达式测试

测试了 `*++cp`, `*cp++`, `++*cp`, `(*cp)++`, `++*++cp`, `++*cp++` 的运算结果和左值性质，以及 `find_char(二级指针)` 的字符串查找。

### Q_2 — find_char (查找字符集合)

在 `source` 中查找第一个匹配 `chars` 集合中任意字符的字符，返回指向该位置的指针。

**核心模式** — 嵌套循环 + 指针副本：
```c
while (*source != '\0') {
    scan = chars;                    // 每次外层前进前，内层指针重置
    while (*scan != '\0') {
        if (*source == *scan)
            return (char *)source;
        scan++;
    }
    source++;
}
return NULL;
```

**关键**: source 每走一步，chars 从头扫一遍。不能两个指针同步走。

### Q_3 — del_substr (删除子串)

删除 `str` 中第一次出现的 `substr`，返回 1（成功）或 0（未找到）。

步骤：
```
1. 用 scan/pat 副本从 str 当前位置开始匹配 substr
2. 完整匹配成功 → 记录匹配起点 match_start
3. 用 write/read 双指针把子串后面的字符整体前移
4. 补 '\0'，返回 1
```

**关键**: 用 `pat = substr` 副本去跑，不动原 `substr`，否则下一轮匹配全乱。

### Q_4 — reverse_string (反转字符串)

原地反转，双指针相向而行：
```c
p_left  = string;                // 指向头
p_right = string + len - 1;      // 指向尾的最后一个字符
while (p_left < p_right) {
    交换 *p_left 与 *p_right;
    p_left++; p_right--;
}
```

**容易犯的错误**: `p_right = string + len - 1`，忘记 `-1` 会导致换到 `'\0'`。

### Q_5 — Eratosthenes 筛选法求质数

```
1. char sieve[SIZE] 全部初始化为 TRUE
2. 0 和 1 标记为 FALSE
3. 从 p=2 开始：
   a. 若 *(sieve+p)==TRUE → p 是质数
   b. 剔除 p 的所有倍数: multiple = 2p → 3p → 4p → ...
   c. p++，重复
4. 当 p*p >= SIZE 时停止
5. 遍历输出 TRUE 的元素
```

**`p*p >= SIZE` 为什么可以停？** — 任何合数 N 必有因子 ≤ sqrt(N)。如果所有 ≤ sqrt(SIZE) 的质数的倍数都已剔除，剩下的全是质数。

优化方向：奇数优化版 — 数组只存奇数，映射 `n = 2k + 3`，空间利用率翻倍。

## 编程注意事项

- **宏末尾不加分号** — `#define TRUE 1` 而非 `#define TRUE 1;`
- **注释用中文，输出用英文** — 避免 Windows 终端中文乱码
- **在 while 循环中 return** — 一旦执行函数立即结束，不会有多个 return
- **`return` 只有一次** — 每个函数调用只产生一个返回值
- **C vs C++ 差异** — 笔记中以 C 为准，遇到 C++ 特性注明区别
