# 编译与链接

## 概述

从 C 源代码到可执行文件，经历四个阶段：

```
预处理 → 编译 → 汇编 → 链接
 hello.c  →  hello.i  →  hello.s  →  hello.o  →  program.exe
```

| 阶段 | 产物 | 做什么 |
|------|------|--------|
| 预处理 | `.i` | 展开 `#include`、`#define` 等 |
| 编译 | `.s` | 把 C 代码翻译成汇编代码 |
| 汇编 | `.o` | 把汇编翻译成机器码（二进制目标文件） |
| 链接 | `.exe` | 把多个 `.o` 拼在一起，解决函数调用地址 |

日常使用时，**编译和汇编合称为"编译"**，所以常说"先编译再链接"。

---

## `-c`：只编译不链接

```bash
gcc -c hello.c          # 生成 hello.o（目标文件，不能直接运行）
```

`-c` 让 GCC 停在 `.o` 这一步，不去找 `main` 函数，不生成可执行文件。

## `-o`：指定输出文件名

```bash
gcc main.c -o app.exe          # 可执行文件叫 app.exe
gcc -c hello.c -o myobj.o     # 目标文件叫 myobj.o
```

不加 `-o` 时默认输出 `a.exe`（可执行文件）或 `源文件名.o`（目标文件）。

---

## GCC 如何决定"编译"还是"链接"

**GCC 看文件后缀来判断该做什么：**

```bash
gcc hello.c             # .c → 编译 + 链接，一步做完
gcc hello.o             # .o → 只链接（已经编译过了，不需要再编译）
gcc main.o hello.o       # .o → 只链接，多个目标文件拼在一起
gcc -c hello.c           # .c → 只编译（-c 拦住了链接这一步）
```

### 本质总结

| 元素 | 作用 |
|------|------|
| `-c` | **喊停**——"编译完就停，不要链接" |
| `-o` | **起名**——"产物叫这个名字" |
| `.c` 后缀 | GCC 知道这是源代码，需要编译 |
| `.o` 后缀 | GCC 知道这是编译过的，直接链接 |

> 💡 不给 `-o` 只做链接（如 `gcc main.o hello.o`）时，产物默认叫 `a.exe`。

## 三种编译方式

示例详见 **`2_basic_conception/Q_1/`**

### 方式一：一步到位（编译 + 链接一起做）

```bash
gcc main.c hello.c -o program.exe
```

适合文件少、改动频繁的小项目。

### 方式二：分步编译再链接

```bash
gcc -c main.c                    # 只编译 main.c → main.o
gcc -c hello.c                   # 只编译 hello.c → hello.o
gcc main.o hello.o -o program.exe # 链接两个 .o → program.exe
```

适合需要理解编译链接过程的学习场景。

### 方式三：增量编译（只重编改过的文件）

```bash
gcc -c hello.c                         # 假设只改了 hello.c
gcc main.o hello.o -o program.exe      # 重新链接
```

大型项目常用：哪个 `.c` 改了只重编那个，其余 `.o` 复用，节省时间。

---

## 批量编译

使用 shell 通配符 `*` 一次性编译当前目录所有 `.c` 文件：

```bash
# 当前目录所有 .c 文件一起编译
gcc *.c -o program.exe
```

`*` 是通配符，`*.c` 匹配"所有以 `.c` 结尾的文件"，shell 自动展开，等价于把文件名一个个写出来。

### 递归查找子目录

```bash
# 查找当前目录及所有子目录下的 .c 文件，一起编译
find . -name "*.c" | xargs gcc -o program.exe
```

### 对比

| 命令 | 范围 |
|------|------|
| `gcc *.c -o app.exe` | 当前目录所有 `.c` |
| `find . -name "*.c" \| xargs gcc -o app.exe` | 递归，含所有子目录 |

> 💡 文件多了以后推荐用 **Makefile**——它能自动判断哪些文件改过，只重编该编的，不用每次都全部重来。这属于本模块后续会学到的内容。

---

## 关键认知

- **编译** = 检查语法，生成目标代码。每个 `.c` 独立编译，互不干涉。
- **链接** = 把目标文件拼成可执行文件，解决函数之间的"找人"问题。
  - `main.c` 调用 `say_hello()`，编译器只记了个"欠条"
  - 链接时找到 `hello.o` 里的 `say_hello()`，把欠条兑现
- 工程越大，**增量编译的优势越明显**：几百个文件只改一个，重编只需几秒。
