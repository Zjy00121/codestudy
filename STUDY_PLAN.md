## 第一阶段：快速上手

> 目标：写起来，边写边学，遇到什么补什么。不追求系统全面，先建立手感。

### 1 快速上手

- **内容**: 基本数据类型、常量和变量、运算符、控制流、函数、数组、字符串、指针、结构体、文件操作
- **重点**: 混个脸熟，不求精通，先能写出能跑的程序
- 状态：[完成]

---

## 第二阶段：基本概念

> 目标：理解 C 程序的构成要素——编译环境、词法规则、代码风格，为后续深入学习打下规范基础。

### 2.1 环境

- **内容**: 编译与链接过程（预处理→编译→汇编→链接）、GCC 工作流程、头文件与库文件搜索路径、集成开发环境
- **重点**: 理解从 `.c` 到 `.exe` 经历的四个步骤，知道每个步骤做了什么
- 状态：[完成]

### 2.2 词法规则

- **内容**: 字符集与编码、标识符命名规则、关键字、常量与字面量、运算符与分隔符、注释（`/* */` 和 `//`）
- **重点**: 合法的标识符是什么样；注释不能嵌套；三字母词（trigraph）的概念
- 状态：[完成]

### 2.3 程序风格

- **内容**: 缩进与空格、大括号风格（K&R / Allman / GNU）、命名规范、函数长度控制、注释原则、代码可读性
- **重点**: 风格可以不同，但必须统一；好的命名比注释更重要
- 状态：[完成]

## 第三阶段：数据

> 目标：透彻理解 C 语言的数据表示、声明与存储机制，这是嵌入式编程的基础——你操作的就是内存里的数据。

### 3.1 基本数据类型

- **内容**: 整型家族（char/short/int/long/long long）、signed vs unsigned、浮点类型（float/double/long double）、各类型在不同平台的大小差异
- **重点**: 嵌入式平台上的类型大小可能和 PC 不同；`sizeof` 是你的尺子
- 状态：[完成]

### 3.2 基本声明

- **内容**: 变量声明语法、初始化 vs 赋值、声明指针、声明数组、声明函数、复杂声明的阅读方法
- **重点**: 学会"从内向外"阅读复杂声明（`int (*arr)[10]` 是什么？）
- 状态：[完成]

### 3.3 typedef

- **内容**: `typedef` 起别名、typedef vs `#define` 的区别、用 typedef 简化复杂声明、结构体 typedef 惯例
- **重点**: typedef 是为类型起新名字，不是创建新类型
- 状态：[完成]

### 3.4 常量

- **内容**: 字面常量、`const` 限定符、`#define` 符号常量、枚举常量（`enum`）、常量表达式
- **重点**: `const` vs `#define` 的区别；`const` 修饰指针的三种位置含义
- 状态：[完成]

### 3.5 作用域

- **内容**: 代码块作用域、文件作用域、函数作用域、原型作用域、变量遮蔽（shadowing）
- **重点**: 同名变量在不同作用域的行为；`-Wshadow` 能帮你发现遮蔽
- 状态：[完成]

### 3.6 链接属性

- **内容**: external / internal / none 三种链接属性、`extern` 和 `static` 对链接属性的影响、头文件中的声明链接问题
- **重点**: 理解"声明"和"定义"的区别；为什么头文件里不能定义变量
- 状态：[完成]

### 3.7 存储类型

- **内容**: 自动存储（栈）、静态存储（静态区）、`auto` / `register` / `static` / `extern` 关键字、寄存器变量的限制
- **重点**: 栈变量的生命周期只在块内；静态变量只初始化一次
- 状态：[完成]

### 3.8 static 关键字

- **内容**: `static` 的三种用法——修饰函数内变量、修饰文件内变量、修饰函数；每种用法的含义
- **重点**: 同一个 `static`，三种语境三种含义，要区分清楚
- **嵌入式关联**: `static` 实现模块级私有变量，保护数据不被外部随意修改
- 状态：[完成]

## 第四阶段：语句

> 目标：系统掌握 C 语言所有语句类型，理解它们的语义和执行流程。

### 4.1 空语句

- **内容**: 只有一个分号 `;` 的语句、什么都不做的占位语句、在循环体为空时的用法
- **重点**: 空语句的副作用——循环条件后面多写一个分号导致循环体变空
- 状态：[ ]

### 4.2 表达式语句

- **内容**: 任何表达式后加分号即构成表达式语句、赋值语句、函数调用语句
- **重点**: 理解"表达式"和"语句"的区别——表达式有值，语句没有
- 状态：[ ]

### 4.3 代码块

- **内容**: `{}` 包裹的复合语句、块内定义的变量作用域、块的嵌套
- **重点**: 代码块可以出现在任何允许单个语句出现的地方
- 状态：[ ]

### 4.4 if 语句

- **内容**: `if` / `else if` / `else`、悬空 else 问题的解决规则、条件表达式的类型转换
- **重点**: else 总是与最近的未匹配 if 结合；用 `{}` 避免歧义
- 状态：[ ]

### 4.5 while 语句

- **内容**: 先判断后执行、循环三要素（初始化/判断/更新）、`break` 和 `continue`
- **重点**: 循环条件中做赋值（`while ((ch = getchar()) != EOF)`）是惯用法
- 状态：[ ]

### 4.6 for 语句

- **内容**: `for (init; cond; update)` 三段式、各段省略的语义、for vs while 的选择
- **重点**: for 适合已知次数的循环；for 的三个表达式都可以省略
- 状态：[ ]

### 4.7 do 语句

- **内容**: 先执行后判断、至少执行一次、vs while 的区别
- **重点**: do-while(0) 在宏定义中的妙用
- 状态：[ ]

### 4.8 switch 语句

- **内容**: `switch` / `case` / `default` / `break`、fall-through、case 必须是常量表达式
- **重点**: 忘记 `break` 导致的 fall-through 是经典 bug；default 不强制但最好写
- 状态：[ ]

### 4.9 goto 语句

- **内容**: `goto` 无条件跳转、标签语法、goto 的正反两面
- **重点**: 跳出深层嵌套、错误处理跳转到统一清理点——这是 goto 为数不多的正当用途
- 状态：[ ]
